## The "Zero-Scroll Contract"

**Rule:** On **1366x768** and **1440x900**, the user must be able to:
1.  Select a Tariff
2.  See the Margin/Provision result
3.  Click "Add to Offer"
**...WITHOUT scrolling the page body.**

If content expands (e.g. huge tariff list), it must scroll **internally** within a panel, not push the summary off-screen.

---

## Layout Regions

### 1. The Shell (Application Frame)
*   **Sticky Header (64px):** Global Navigation, User Profile.
*   **Sticky Sub-Header (48px):** Breadcrumbs, Step Indicator, View Mode.
*   **Sidebar (Collapsible):** Navigation (Home, History, Settings).

### 2. The Cockpit (Work Area)
*   **Layout:** CSS Grid / Flex with sticky exclusions.
*   **Left Rail (Input):** `flex-1`, `overflow-y-auto`.
    *   Internal components (Tariff List) should consolidate height.
    *   Accordion approach allows progressive disclosure to save vertical space.
*   **Right Rail (Output):** `width: 320px`, `sticky`, `top: 112px`.
    *   Contains: `LiveSummary`, `ValidationStatus`, `PrimaryCTA`.
    *   **CRITICAL:** Must never be taller than viewport height minus headers.

### 3. Mobile Adaptation (< lg)
*   **Right Rail:** Disappears.
*   **Bottom Sheet:** "Live Summary" acts as a sticky footer (collapsed 60px) that expands on tap.
*   **Primary CTA:** Always visible in the sticky footer.
